Read Later — Privacy Policy

Privacy Policy

How the Read Later Chrome extension handles your data.

Last updated: July 14, 2026

The short version: Read Later does not collect, transmit, sell, or share any personal data. Everything it saves stays on your own device — as bookmarks in your browser. No accounts, no analytics, no servers, no tracking.

1. Overview

Read Later ("the extension") is a lightweight read-it-later tool that lets you quickly save the web page you are currently viewing into a dedicated bookmarks folder, and manage those saved pages so you can return to them later. This policy explains what the extension does and does not do with any data involved in that process.

This policy applies to the Read Later extension distributed via the Chrome Web Store. By installing and using the extension, you are interacting with this policy.

2. Information We Collect

The extension does not collect any personal information. It does not require you to create an account, sign in, or provide any identifying details.

Specifically, the extension does not collect, transmit, or process:

  • Your name, email address, or any contact details.
  • Your browsing history or the pages you visit.
  • Authentication credentials, tokens, or payment information.
  • Device identifiers, advertising identifiers, or precise/coarse location.
  • Personal communications or the content of web pages.

The only data the extension reads is the title and URL of the active tab, and only at the exact moment you choose to save a page for later reading. That information is then stored as a bookmark — it is never sent anywhere outside your browser.

3. Browser Permissions & How They Are Used

The extension requests a small set of browser permissions. Each is used solely to deliver its core function of saving and managing pages for later reading. Below is the purpose of each.

  • bookmarks Core to the extension. Saved pages are stored as bookmarks inside a dedicated "read later" folder managed by the extension. The extension reads, creates, moves, updates, and removes bookmarks only within its own folder. It does not access, read, or modify the rest of your bookmarks.
  • tabs Used to read the title and URL of the currently active tab when you choose to save a page. Only the active tab's title and URL are read at the moment of saving; no tab content is read and no browsing history is collected.
  • contextMenus Used to add a "read later" item to the browser's right-click context menu on pages, so you can save the current page without opening the popup.
  • notifications Used to show a small success/warning notification when a page is saved, a duplicate is detected, or a post is removed, giving you immediate feedback on the action you just performed.
  • storage Used to persist your extension settings (such as the bookmarks folder name) locally, so your configuration survives browser restarts. No personally identifiable information is stored; settings are kept on your device only.

4. How Data Is Stored

All data the extension creates lives on your own device, inside your browser profile:

  • Saved pages are stored as ordinary bookmarks in the browser's native bookmarks system, within the "read later" folder.
  • Settings are stored in the browser's local extension storage.

The extension has no backend server and no database under its control. Your data never leaves your browser. Because saved items are normal bookmarks, you can view, export, back up, or delete them at any time using the browser's built-in bookmarks manager — uninstalling the extension does not delete the bookmarks it created.

5. Third-Party Services

The extension does not integrate with, send data to, or rely on any third-party service. There are no analytics providers, advertising networks, tracking SDKs, or external APIs used. No external servers receive any data from the extension.

6. Remote Code

The extension does not load, execute, or reference any remote code. All JavaScript and assets are bundled within the extension package at build time. The extension does not fetch or evaluate code from any external server, CDN, or URL at runtime.

7. Children's Privacy

The extension is intended for a general audience and does not knowingly collect any data from anyone, including children. Because no personal data is collected at all, no special protections for children's data are required — there is nothing to protect.

8. Changes to This Policy

If the extension's data practices change in a way that materially affects this policy, this page will be updated and the "Last updated" date above revised. Because the extension does not collect personal data, material changes to data handling are not expected.
